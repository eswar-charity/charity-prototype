UI/UX Polish & Accessibility Pass

Perform a comprehensive UI/UX refinement across the application while preserving all existing functionality, business logic, and navigation. The goal is to produce an investor-demo-ready interface with consistent styling, improved readability, and better accessibility.

Layout & Spacing
Ensure consistent spacing throughout the application.
Increase whitespace where needed to improve readability, especially in comments and content-heavy sections.
Icons
Audit all icons across the application.
Ensure icons are visually consistent in style, size, weight, alignment, and color.
Replace inconsistent icons with a unified icon set where necessary.
Cards
Every card should have clearly defined boundaries.
Use consistent border radius, elevation, and padding.
Ensure cards are visually separated from surrounding content.
Pills / Tags
Make pills more prominent and easily scannable.
Maintain consistent height, padding, border radius, typography, and spacing.
Color System

Implement a consistent design system using multiple shades of blue:

Primary Blue – Primary CTAs and key actions.
Secondary Blue – Secondary emphasis and selected states.
Tertiary Blue – Informational elements and self chat bubbles.

Use complementary colors where appropriate instead of relying on a single blue tone.

Selected States
Selected tabs should use a distinct secondary blue.
Active navigation items should be clearly distinguishable from inactive ones.
Chat UI
Use the tertiary blue for self chat bubbles.
Improve spacing between chat messages.
Make the comment section feel more open and comfortable to read.
Forms & Controls

The following components should not use blue by default:

Text fields
Media upload controls
Secondary CTAs
One-time action buttons

Use neutral colors for these elements unless they represent a primary action.

Event Feed
Reduce the amount of text displayed in each post body.
Show concise previews with appropriate truncation.
Preserve the full content for the detailed view.
Review Cards
Replace the existing review card colors with complementary colors that fit the overall design system.
Ensure sufficient contrast and visual distinction.
Accessibility

Run a complete accessibility audit and fix all issues, including but not limited to:

Color contrast
Keyboard navigation
Focus indicators
ARIA labels
Form labels
Semantic HTML
Touch target sizes
Screen reader compatibility
Responsive behavior

Pay special attention to the Event Details page and ensure it passes accessibility best practices.

Content Cleanup
Remove all unnecessary emojis from the UI.
Keep the interface professional and consistent.
Constraints
Do not change business logic.
Do not modify user flows or navigation.
Do not redesign layouts beyond the improvements listed above.
Preserve all existing functionality while improving the visual polish and usability.
Expected Outcome

The application should feel polished, visually consistent, accessible, and ready for an investor demonstration, with a cohesive design system, improved readability, and a professional user experience.
